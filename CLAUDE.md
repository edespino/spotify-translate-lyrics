# CLAUDE.md

Context for working on this repo.

## What this is

Personal web app: shows time-synced Spanish lyrics for the currently playing Spotify track with a line-aligned English translation, Spotify-style scrolling. Two processes started by one `npm run dev` (concurrently): a Vite + React + TypeScript frontend and a small Express + TypeScript server run with tsx.

## Layout

- `src/spotify.ts` Spotify OAuth (Authorization Code with PKCE, no client secret, tokens in localStorage, silent refresh). Polls /me/player/currently-playing every 3s, backs off to 10s on 429. seekTo PUTs /me/player/seek with the same 401-refresh handling; 403/404 (no active device or restricted context) throw SeekUnavailableError, which App shows as a brief notice. Seek requires the user-modify-playback-state scope; sessions connected before it was added must reconnect once.
- `src/sync.ts` PositionTracker interpolates playback position between polls with performance.now(); a poll that disagrees with interpolation by more than 1.5s (seek) or a play/pause flip re-anchors. nudge(positionMs, nowMs) re-anchors immediately after a locally initiated seek (line replay) so the highlight snaps without waiting for a poll; App also triggers an immediate poll to reconcile. findActiveLine is a binary search over line timestamps.
- `src/lrc.ts` LRC parser: handles minutes over 9, 1 to 3 digit fractions, multiple timestamps per line, skips malformed lines, keeps empty text lines.
- `src/lyrics.ts` LRCLIB client (https://lrclib.net/api/get, CORS is open, called from the browser). Result kinds: synced, plain, instrumental, none.
- `src/detect.ts` client-side English detection on the full joined lyric text, before any /api/translate call. Pure stopword and character frequency heuristic (English vs Spanish stopword hit rates plus accented characters and inverted punctuation), no network, no library. Biased: only confidently English text skips translation; uncertain or mixed (Spanglish) text translates as before. English songs render read-only in a single centered pane (src/translation.ts holds the fetch-if-needed flow that returns null without any network call in that case).
- `src/App.tsx` state machine: auth, poll loop, per-track lyrics + translation loading, requestAnimationFrame loop for the active line.
- `src/components/LyricsView.tsx` both languages render in ONE scroll container as two-column grid rows, so lines stay vertically aligned by construction; the active row is smooth-scrolled to a 32% anchor with a top clamp (long jumps and reduced-motion users get an instant snap). The Spanish column is right-aligned in dual-pane mode so pairs meet at the central gutter. Rows before the active line get a `past` class and render brighter than upcoming rows. Enlarge and the active pop scale CELLS, not the row, so column alignment survives. Rows are focusable (Enter toggles enlarge, F2 or e edits the Spanish cell), carry list/listitem roles, aria-current on the active row, and per-language lang attributes. Double-click edits a cell, click enlarges a row. While translating, EN cells show shimmering skeleton bars (widths cycle 45/60/75% by row; shimmer only under no-preference motion); on translation error they show a dim placeholder instead, and the header shows a prominent retry button. Per-line replay: synced rows with text (not the empty ♪ placeholders, never plain lyrics) get an absolutely positioned replay button in the row's left padding, visible on the active row and on hover/focus, with its own click target (stopPropagation) so row click/double-click/Enter behavior is untouched; `r` on a focused row replays that row (stopPropagation so the global handler stays quiet), `r` globally replays the active line, and both are guarded against text inputs. English single-pane mode keeps the replay affordance. Pure per-row logic (phase classification, EN cell state, key mapping including replay, replay eligibility, typing-target guard, scroll behavior) lives in `src/components/lyricsRow.ts` and is unit tested.
- `src/components/EmptyState.tsx` shared screen for the non-lyric states (lyrics loading, none, instrumental, lyrics-service error, nothing playing): ~200px album art (dim music-note block when no artwork URL), track title, status line, centered in the lyrics area. The lyrics-service error adds a Retry pill wired to App's loadLyrics, which re-runs the LRCLIB fetch for the current track without waiting for a track change. State-to-message mapping, retryability, and skeleton width cycling are pure functions in `src/components/emptyState.ts`, unit tested.
- `server/app.ts` Express app factory (takes a provider and data dir, which is what the integration tests use).
- `server/translator.ts` provider interface plus the Gemini implementation (@google/genai, model from GEMINI_MODEL, default gemini-flash-lite-latest, JSON array in and out). translateLines enforces the N-in N-out contract: retry the batch once, then fall back to chunks of 20 lines, each strictly validated. A 429 waits for the provider's suggested retry delay (bounded attempts). Any chunk failure fails the whole translation with TranslationFailedError; the server answers 502 and caches nothing. Original text is never returned as a translation.
- `server/cache.ts` disk cache at data/translations/<trackId>.json. Overrides (editedEs/editedEn) always win; retranslate never overwrites en on a line with editedEn.
- Title translation: translateLinesWithTitle prepends the song title as the first line of the same Gemini batch (no extra request; the N-in N-out count includes it; in the chunked fallback the title rides in the first chunk). The result is stored as titleEn on the cache entry and returned by /api/translate. Cache files written before titleEn existed lazily backfill only titleEn on the next cached response using one title-only provider request; failure is ignored so cached lyrics still return, and the next play retries. Retranslate refreshes titleEn. NowPlaying shows titleEn inline after the original title as muted parenthesized text, suppressed when it matches the original case-insensitively (src/translation.ts translatedTitle) and never in English single-pane mode. The title is not editable through the override system.
- API: POST /api/translate, POST /api/retranslate, POST /api/override, POST /api/override/reset, GET /api/translations/:trackId. Frontend reaches it through the Vite proxy at /api.

## Rules

- `data/` is user data derived from copyrighted lyrics. Never commit it, never weaken the .gitignore entry.
- GEMINI_API_KEY stays server-side only. Never expose it to the browser or prefix it with VITE_.
- Timestamps are never editable, only the text of a line.
- No em dashes or en dashes anywhere in this repo, including docs and commit messages. ASCII punctuation only.
- Commit messages are terse and conventional, no AI attribution of any kind.

## Checks

`npm test`, `npm run typecheck`, `npm run lint` must all pass. Tests are Vitest; server integration tests use supertest with a mocked provider and a temp data dir.
